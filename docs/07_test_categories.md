# 07 — 五大类测试边界、强度等级与停止条件

> 对应 docx 第 6、13 节 + xlsx "字典" sheet。每类用例只关注**自己应该关注的指标**，不要越界判断。

---

## 五大类

| 类前缀 | 一级分类 | 目标 | 关键验收点 | 不要越界 |
|---|---|---|---|---|
| `SOC-TEL-*` | 遥测采集与基础链路 | 验证日志、规则、转发、case 基础链路 | 字段完整、转发成功、敏感值脱敏 | 不要判断"AI 研判是否正确"——那是 WF 的事 |
| `SOC-ATT-*` | MITRE 攻击场景 | 覆盖真实攻击阶段 | 检测、关联、ATT&CK 归类、响应建议 | 不要验证"是否未授权登录"——那是 AI 的事 |
| `SOC-AI-001~014` | AI 平台/API 安全 | 身份、租户、WebIDE/WattBot 临时入口 | 未授权访问为 0、跨租户泄露为 0 | 不要试压力/洪泛——那是 Socket/L3 的事 |
| `SOC-AI-015~030` | Agent/Socket 运行时安全 | 长连接、SSE、工具调用、Prompt 注入 | 会话收敛、限流、幂等、审批有效 | 不要去验证"日志是否进 Wazuh"——那是 TEL 的事 |
| `SOC-WF-*` | SOC 工作流与 AI 研判 | alert → finding → case → AI → response | 攻击链还原、低幻觉、可审批可回滚 | 不要去重复 ATT 的攻击模拟——直接用现有 findings |

## 强度等级 L0~L4（xlsx 字典 sheet）

| 强度 | 含义 | 可执行环境 |
|---|---|---|
| **L0** | 被动核查（只读、不改变系统状态） | 生产低峰也可 |
| **L1** | 非破坏（产生少量受控日志，不改变业务） | 生产低峰也可 |
| **L2** | 受控滥用（少量爆破、越权尝试） | 预生产 / 隔离租户 |
| **L3** | 压力/对抗（洪泛、SSE 重连风暴、慢消费） | 专用压测/克隆环境 |
| **L4** | 克隆环境破坏性 / 长稳 | 仅克隆环境 |

**本仓库 20 条 P0 全部在 L1~L2 范围内**。L3/L4 的 DoS / 洪泛类（如 SOC-ATT-046~048、SOC-AI-015~019 中的部分）未纳入首版。

## 停止条件（docx 第 13 节）

下列任一情况出现，**立即停止相关测试并进入缺陷处理**：

1. 未授权 101 (WebSocket Switching) / 200 (敏感数据 200) 出现
2. 跨租户数据泄露
3. 长期凭据泄露
4. 可信头（X-User-*、X-Workspace-*）可伪造并被服务端采信
5. ticket / session 可重放
6. 生产错误率、延迟、连接数、CPU、内存、fd、队列长度达到预设红线（L2 以上停止）
7. 自动响应影响非测试主体 / 无法确认回滚路径 / 审批链路失效
8. 证据链丢失 / 时间不同步 / 无法关联 case_id 与 run_id —— **本轮不得作为正式验收证据**

## 风险控制

每个用例文件顶部都用 pytest mark 标识所需触达面：

```python
pytestmark = [pytest.mark.p0, pytest.mark.att, pytest.mark.needs_ssh, pytest.mark.needs_wazuh]
```

- `needs_ssh` → 缺凭据自动 skip
- `needs_wazuh` → 缺凭据自动 skip
- `destructive` → 默认 skip，需显式 `-m destructive` 才跑（本仓库暂未启用）

这种 marker 体系保证：**新人在不知道边界的情况下，最多只能跑到 L1~L2 的用例**。
